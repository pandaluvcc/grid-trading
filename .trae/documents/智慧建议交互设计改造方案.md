# 智慧建议交互设计改造方案（个人版）

> 版本：v1.2  
> 日期：2026-03-06  
> 目标：优化智慧建议的交互体验，提升操作效率  
> 适用场景：个人自用系统

---

## 一、当前设计问题分析

### 1.1 现有设计

当前智慧建议的交互流程：

```
首页策略卡片 → 点击进入详情页 → 查看SmartSuggestion组件 → 手动输入价格 → 获取建议
```

**主要问题：**

| 问题 | 描述 | 影响 |
|------|------|------|
| **入口过深** | 建议信息隐藏在详情页内，用户需要多次点击才能查看 | 用户容易错过重要建议 |
| **信息层级不清** | 风险提示、建议操作、市场状态混在一起 | 用户难以快速获取关键信息 |
| **被动触发** | 需要用户手动输入价格才能获取建议 | 增加用户操作成本 |
| **缺乏消息中心** | 没有统一的消息管理入口 | 建议无法集中查看和管理 |
| **视觉提示弱** | 首页只有小图标提示，不够醒目 | 用户感知度低 |

### 1.2 用户痛点

1. **不知道有建议**：首页提示不明显，用户可能不知道有操作建议
2. **查看成本高**：需要进入详情页才能看到具体建议
3. **操作繁琐**：需要手动输入价格、刷新才能获取最新建议
4. **信息过载**：建议列表信息量大，缺乏优先级区分
5. **无法批量处理**：多个策略的建议无法统一管理

## 二、主流APP设计参考

### 2.1 券商APP设计趋势

根据市场调研，主流券商APP的设计特点：

| APP | 核心设计理念 | 值得借鉴 |
|-----|-------------|---------|
| **华泰AI涨乐** | AI原生交易APP，以交易场景为核心 | 智能决策与交易无缝衔接 |
| **银河证券** | 卡片式瀑布流，场景化+模块化 | 信息分层清晰，可折叠卡片 |
| **同花顺** | 买卖点提醒，条件单功能 | 主动提醒，多条件触发 |
| **富途牛牛** | 极致用户体验，条件单丰富 | 简洁操作流程，一键执行 |
| **老虎证券** | 互联网产品思维，年轻化设计 | 社交化、游戏化元素 |

### 2.2 消息系统设计原则

根据产品经理社区的最佳实践：

```
消息传播效率最高（获取、处理、信息传达、用户反馈等效率）
避免产生骚扰（噪音、频繁提示）
```

**关键设计模式：**

1. **消息分类**：通告、提醒、私信等
2. **消息合并**：同类型消息合并，减少打扰
3. **处理状态**：已读/未读统一管理
4. **卡片式设计**：信息收纳，视觉层级清晰

### 2.3 通知类型对比

| 类型 | 特点 | 适用场景 | 当前使用 |
|------|------|---------|---------|
| **Badge（标记）** | 图标右上角数字，不紧急 | 未读消息数 | ✅ 首页策略卡片 |
| **Banner（横幅）** | 顶部临时提示，自动消失 | 即时提醒 | ❌ 未使用 |
| **Alert（弹窗）** | 强打断，需用户操作 | 重要紧急提醒 | ❌ 未使用 |
| **Toast（轻提示）** | 纯告知，不需要处理 | 操作反馈 | ✅ 已使用 |
| **通知栏** | 集中处理，可汇总 | 消息中心 | ❌ 未使用 |

```vue
        {{ suggestion.reason }}
      </div>
    </div>
    
    <div class="card-footer">
      <el-button size="small" @click="$emit('viewGrid', suggestion)">
        查看网格
      </el-button>
      <el-button 
        size="small" 
        :type="suggestion.type === 'BUY' ? 'danger' : 'success'"
        @click="$emit('execute', suggestion)"
      >
        {{ suggestion.type === 'BUY' ? '执行买入' : '执行卖出' }}
      </el-button>
    </div>
  </div>
</template>
```
